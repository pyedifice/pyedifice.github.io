edifice.App

edifice.App#

class edifice.App(root_element, inspector=False, create_application=True, application_name=None, qapplication=None)[source]#

Bases: object

The main application object.

To start the application, call the App.start() method:

App(MyRootElement()).start()

If you want application to be a normal application in an operating system window, then the top Element rendered by MyRootElement must be a Window.

If you just want to create a widget (that you’ll integrate with an existing codebase), call the App.export_widgets() method:

widget = App(MyRootElement()).export_widgets()

This widget can then be plugged into the rest of your application, and there’s no need to manage the rendering of the widget – state changes will trigger automatic re-render without any intervention.

Parameters:

  • root_element (Element) – the root Element of the application. It must render to an instance of Window or some other RootElement

  • inspector (bool) – whether or not to run an instance of the Edifice Inspector alongside the main app. Defaults to False

  • create_application (bool) – (default True) whether or not to create an instance of QApplication. Usually you want to use the default setting. However, if the QApplication is already created (e.g. in a test suite or if you just want Edifice to make a widget to plug into an existing Qt application), you can set this to False.

  • application_name (Optional[str]) – DEPRECATED the Qt application name to set when creating a new QApplication. This option is only relevant if create_application is True.

  • qapplication (Optional[QApplication]) – (default None) The QtWidgets.QApplication. If you do not provide one, it will be created for you.

export_widgets()[source]#

Exports the Qt widgets underlying the Edifice Element.

Depending on how the root component is defined, either one or multiple widgets are returned (for example, if your root component returns a list of Views, export_widgets will return a list of widgets). These widgets are still managed by Edifice: they will still benefit from full reactivity and state consistency. You can mount these widgets to your pre-existing Qt Application this way:

# Suppose parent_widget is defined in Qt code.
app = edifice.App(MyAwesomeElement())
widget = app.export_widgets()
widget.setParent(parent_widget)

start_loop()[source]#

Start the application event loop manually.

A context manager alternative to App.start() which allows access to the application’s qasync.QEventLoop.

The asyncio event loop uses the QEventLoop as the current event loop. You can also access the asyncio current event loop in the usual way with asyncio.get_running_loop().

When you use start_loop(), you must explicitly run the event loop, for example with run_forever(). When the user closes the App it will stop the event loop, see QApplication.exit_().

In this example, we add a Unix SIGINT handler which will also stop the event loop.

Example:

app = edifice.App(MyAppElement())
with app.start_loop() as loop:
    loop.add_signal_handler(signal.SIGINT, loop.stop)
    loop.run_forever()
